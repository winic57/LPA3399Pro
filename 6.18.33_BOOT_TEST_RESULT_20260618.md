## ❌ 两个关键问题

### 问题 1：时钟驱动注册失败（致命）

```
Line 297: clk: couldn't get clock 5 for /clock-controller@ff760000
Line 298: rockchip_clk_of_add_provider: could not register clk provider
Line 302: Failed to get pclk for 'rk_timer'
```

**原因**: vendor DTB 使用 Rockchip 私有时钟 ID 编码（compatible = "rockchip,rk3399-cru"），主线内核 6.18.33 的时钟驱动（`clk-rockchip-rk3399.c`）期望的 DT binding 格式与 vendor DTB 不兼容。

**影响**: 时钟驱动注册失败 → 几乎所有外设时钟不可用 → 系统无法继续启动。

### 问题 2：DTB chosen 节点覆盖了 extlinux APPEND

| 参数 | extlinux.conf 设置 | 内核实际收到 | 来源 |
|---|---|---|---|
| root | UUID=d52f7f8b-... | PARTUUID=614e0000-0000 | DTB chosen |
| console | ttyS2,1500000 | ttyFIQ0 | DTB chosen |
| earlycon | uart8250,...,1500000n8 | uart8250,...（去掉波特率） | DTB chosen |

**原因**: vendor DTB 的 `/chosen` 节点中 embedded 了 `bootargs` 属性，内核优先使用 DTB 中的 bootargs，覆盖 U-Boot 传递的参数。

**影响**: 
- console=ttyFIQ0 是 vendor 内核专用设备（fiq debugger），主线 6.18.33 内核没有编译 → 串口控制台在 0.007s 后断开
- root=PARTUUID=614e0000-0000 指向错误的分区

## 根因：主线内核 6.18.33 vs vendor DTB 不兼容

vendor DTB (`rk3399pro-neardi-linux-lc110-base.dtb`) 是为 Rockchip 4.4.194 vendor 内核制作的。主线内核 6.18.33 的设备驱动（时钟、pinctrl、interrupts 等）期望的 DT binding 格式与 vendor DTB 差异太大：

| 组件 | vendor DTB | 主线内核期望 | 兼容性 |
|---|---|---|---|
| 时钟控制器 | `rockchip,rk3399-cru` (私有 ID) | `rockchip,rk3399-cru` (DT binding) | ❌ clock ID 不匹配 |
| UART | `fiq_debugger` / `ttyFIQ0` | `dw-apb-uart` / `ttyS2` | ❌ console 不匹配 |
| bootargs | chosen 节点 embedded | U-Boot 传递 | ❌ 被 DTB 覆盖 |
| GMAC | `rockchip,rk3399-gmac` | `rockchip,rk3399-gmac` | ⚠️ 待验证 |

---

## 结论

**6.18.33 主线内核 + vendor DTB 路线不可行。**

即使修复了 SD 卡启动问题（extlinux.conf），内核也无法正常初始化核心子系统（时钟、定时器），因为 vendor DTB 与主线内核的 DT binding 不兼容。

---

## 下一步建议

### 方案 A：使用主线 DTB（推荐）✅

主线内核源码中包含 RK3399 的标准 DTB（`arch/arm64/boot/dts/rockchip/rk3399.dtsi`），这些 DTB 与主线内核完全兼容。

**操作**: 从 6.18.33 内核编译产物中提取 DTB，替换 SD 卡上的 vendor DTB。

**风险**: 主线 DTB 可能不支持 LPA3399Pro 板卡的特殊外设（NPU、双摄像头等），但基本功能（CPU、USB、网络、SD 卡）应该可用。

### 方案 B：切换到 Path C — vendor 4.4.194 内核

板载 eMMC 已经有 vendor 4.4.194 内核，与 vendor DTB 完全兼容，GMAC 驱动也在（虽然 DMA 有硬件缺陷）。

**操作**: 不使用 SD 卡，直接从 eMMC 启动。使用 USB 以太网适配器替代板载 GMAC。

### 方案 C：修复 vendor DTB 的时钟绑定

手动修改 vendor DTB 中的时钟控制器节点，使其兼容主线内核。

**风险**: 工作量巨大，需要逐个修改 clock ID 映射，容易出错。
