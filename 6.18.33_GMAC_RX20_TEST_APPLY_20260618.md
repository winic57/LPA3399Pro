# 6.18.33 GMAC rx_delay=0x20 测试应用记录

日期: 2026-06-18
目的: 在 `gmac-test.dtb` 基础上仅将 `rx_delay` 改为 `0x20`，继续定位 DMA reset 失败问题。

## 修改位置

当前运行系统中的 boot 分区挂载点：

```text
/mnt/sdboot
```

## 新 DTB

```text
/mnt/sdboot/dtb/rockchip/gmac-rx20-test.dtb
```

## 修改命令

```bash
cp -a /mnt/sdboot/dtb/rockchip/gmac-test.dtb /mnt/sdboot/dtb/rockchip/gmac-rx20-test.dtb
fdtput -t x /mnt/sdboot/dtb/rockchip/gmac-rx20-test.dtb /ethernet@fe300000 rx_delay 0x20
```

## 验证

```text
fdtget -t x /mnt/sdboot/dtb/rockchip/gmac-rx20-test.dtb /ethernet@fe300000 rx_delay
20
```

## extlinux 切换

已将：

```text

```

改为：

```text
FDT /dtb/rockchip/gmac-rx20-test.dtb
```

```text
/mnt/sdboot/extlinux/extlinux.conf.gmac-rx20-test.bak
```
