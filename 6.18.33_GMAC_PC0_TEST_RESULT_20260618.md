# 6.18.33 GMAC reset GPIO PC0 测试结果

日期: 2026-06-18
测试 DTB: `/dtb/rockchip/gmac-pc0-test.dtb`
测试变量: 仅将 `snps,reset-gpio` 从 `gpio3 PB7 active-low` 改为 `gpio3 PC0 active-low`，保持 `rx_delay=0x11`。

## 启动结果

系统成功启动到 root 自动登录。

启动阶段确认加载：

```text
Retrieving file: /dtb/rockchip/gmac-pc0-test.dtb
```

GMAC probe 正常：

```text
rk_gmac-dwmac fe300000.ethernet: clock input or output? (input).
rk_gmac-dwmac fe300000.ethernet: TX delay(0x28).
rk_gmac-dwmac fe300000.ethernet: RX delay(0x11).
rk_gmac-dwmac fe300000.ethernet: init for RGMII
YT8521 Gigabit Ethernet stmmac-0:00: attached PHY driver
YT8521 Gigabit Ethernet stmmac-0:03: attached PHY driver
```

## 失败点

eth0 open 阶段仍失败：

```text
rk_gmac-dwmac fe300000.ethernet eth0: Register MEM_TYPE_PAGE_POOL RxQ-0
rk_gmac-dwmac fe300000.ethernet eth0: PHY [stmmac-0:00] driver [YT8521 Gigabit Ethernet] (irq=POLL)
rk_gmac-dwmac fe300000.ethernet eth0: Failed to reset the dma
rk_gmac-dwmac fe300000.ethernet eth0: stmmac_hw_setup: DMA engine initialization failed
rk_gmac-dwmac fe300000.ethernet eth0: __stmmac_open: Hw setup failed
```

## 结论

单独把 reset GPIO 从 PB7 改为 PC0 不能解决 DMA reset failed。

目前已排除的单变量：

1. `rx_delay = 0x20`：仍失败；
2. `snps,reset-gpio = gpio3 PC0`：仍失败。

下一步更应关注：

- GMAC 时钟方向 `clock_in_out`；
- assigned clock parent / `SCLK_RMII_SRC`；
- `clk_mac_ref` 是否真正来自 PHY 125MHz；
- 主线 `dwmac-rk.c` 与 vendor 4.4 初始化差异。
