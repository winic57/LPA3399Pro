# 6.18.33 GMAC 可试 patch 草案说明（2026-06-18）

## 1. patch 文件

已生成草案 patch：

```text
/mnt/sdb3/LPA3399Pro/patches/0001-rk3399pro-gmac-debug-and-rxc-workaround.patch
```

## 2. 这版 patch 的定位

这不是“最终正确修复”，而是 **第一版可试验补丁**，目标是：

1. 增加 RK3399 GMAC bring-up 的可观测性；
2. 针对 LPA3399Pro + YT8521 外置 PHY 的特征，尝试一个保守的 clock/reset workaround；
3. 让下一位 agent 可以直接进入“编译-刷入-验证”阶段，而不是从零开始写 patch。

## 3. patch 主要做了什么

### 3.1 增加 RK3399 GRF readback 日志

在 `rk3399_set_to_rgmii()` 后打印：

- `GRF_SOC_CON5`
- `GRF_SOC_CON6`
- `tx_delay`
- `rx_delay`

目的：确认 RGMII mode / delay 配置是否真的写进 GRF。

### 3.2 增加外置 PHY reset helper

新增 `rk_gmac_external_phy_reset()`，对外置 PHY 做：

- assert
- 10ms 延时
- deassert
- 50ms 延时

目的：补一轮“时钟/GRF/供电都就绪后的二次 PHY reset”。

### 3.3 在 RK3399Pro 外置 RGMII PHY 场景强制 `clock_input=true`

在测试路径里，对：

- 非 integrated PHY
- RGMII 系列接口

强制：

```c
bsp_priv->clock_input = true;
```

目的：

- 避免某些 output 模式试验把 MAC 端带到不稳定路径；
- 更接近当前板子的 vendor/实测主路径（外置 PHY 给 RXC）。

### 3.4 在 `phy_power_on()` + GRF 配置之后再做一次 PHY reset

在 `rk_gmac_powerup()` 中：

- clocks enable
- `set_to_rgmii()`
- `phy_power_on(true)`
- 然后对外置 PHY 再 reset 一次
- 再等待 20~30ms
- 最后返回给 stmmac

目的：解决可能存在的“DMA 初始化前，PHY/MAC 时钟域没有完全稳定”的问题。

### 3.5 增加 powerup 起止日志

打印：

- `phy_iface`
- `clock_input`
- `integrated_phy`
- `tx_delay`
- `rx_delay`

便于 TTL 直接判断补丁路径是否真的执行。

## 4. 为什么先做这个 patch

因为当前已有证据表明：

- DTB 方向基本已经试到头；
- `ethtool -d eth0` 显示 DMA 寄存器前后几乎不变；
- 更像是 clock/reset/init sequence 没真正到位。

所以这版 patch 采用“**最小介入 + 最大观测性**”策略，而不是一上来大改 stmmac core。

## 5. 这版 patch 的风险与局限

### 风险

1. 强制 `clock_input=true` 可能与某些 output 测试 DTB 冲突；
2. 二次 PHY reset 可能对个别板型并不适合；
3. 如果根因在 AHB reset / DMA core reset / runtime PM，而不是 PHY 时序，这版 patch 可能无效。

### 局限

它没有修改：

- stmmac core DMA reset 流程
- AHB reset 控制逻辑
- runtime PM 的 bus clk 开关路径
- clk_mac_ref / clk_mac_refout 的 deeper handling

因此如果这版 patch 无效，下一版应进一步转向：

1. `stmmac_hw_init()` / DMA reset 周边 debug patch
2. AHB/stmmac reset sequence patch
3. runtime PM / bus clk keep-on workaround

## 6. 下一位 agent 的推荐动作

### 6.1 先把 patch 应到实际 6.18.33 源码树

目标文件：

```text
drivers/net/ethernet/stmicro/stmmac/dwmac-rk.c
```

### 6.2 编译出可启动内核/模块

编译后替换到当前 SD 测试环境，保持 DTB 仍使用：

```text
/dtb/rockchip/gmac-phyhandle-pll-test.dtb
```

### 6.3 TTL 重点看这些新日志

```text
rk3399 rgmii applied: GRF_SOC_CON5=...
external PHY reset (post-grf-post-power) begin/end
powerup start: ...
powerup done: ...
```

### 6.4 验证成功标准

至少要看到：

1. `ip link set eth0 up` 不再卡 20 秒；
2. 不再报：
   - `Failed to reset the dma`
   - `stmmac_hw_setup: DMA engine initialization failed`
3. `dmesg` 出现 link up / carrier 相关日志；
4. `dhclient eth0` 能拿到 IP。

## 7. 如果这版 patch 失败，下一版补丁建议

优先顺序：

### Patch v2 建议

1. 在 `stmmac_hw_init()` / DMA reset 前后打印 DMA bus mode / status 寄存器；
2. 对 `stmmac_ahb_rst` + `stmmac_rst` 加更激进 reset 序列；
3. 对 runtime PM / `stmmac_bus_clks_config()` 做 keep-on workaround；
4. 必要时在 RK3399 path 里显式启用 `clk_mac_refout` / 增加 parent select debug。

## 8. 一句话总结

> 这版 patch 是“**先把 RK3399Pro 外置 YT8521 的 clock/reset 观测与保守 workaround 做起来**”的第一版可试草案，适合直接交给后续 agent 进入编译验证。
