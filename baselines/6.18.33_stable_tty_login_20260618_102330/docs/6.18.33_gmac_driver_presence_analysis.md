# 6.18.33 有线网卡问题与 GMAC 驱动是否缺失的核查结论

## 结论

当前有线网卡 `eth0` 不存在，**不是因为内核编译缺少 GMAC 驱动 `dwmac-rockchip`**。

实际原因是当前稳定 DTB 中：

```text
/proc/device-tree/ethernet@fe300000/status = disabled
```

所以平台设备没有创建，驱动没有 probe，自然不会出现 `eth0`。

## 1. 编译日志证据

`/mnt/sdb3/LPA3399Pro/rk3399pro_kernel/compile.log` 中明确显示 stmmac 和 Rockchip DWMAC 被编译：

```text
CC      drivers/net/ethernet/stmicro/stmmac/stmmac_main.o
CC      drivers/net/ethernet/stmicro/stmmac/stmmac_platform.o
CC      drivers/net/ethernet/stmicro/stmmac/dwmac-rk.o
AR      drivers/net/ethernet/stmicro/stmmac/built-in.a
```

其中 `dwmac-rk.o` 进入 `built-in.a`，说明它是内建进内核，而不是 `.ko` 模块。

## 2. 编译计划证据

`/mnt/sdb3/LPA3399Pro/rk3399pro_kernel/compile_plan.md` 中也记录了目标配置：

```text
CONFIG_DWMAC_ROCKCHIP=y
CONFIG_STMMAC_ETH=y
CONFIG_STMMAC_PLATFORM=y
```

## 3. 运行时 TTL 证据

通过 TTL 在运行系统中检查 `/proc/config.gz`：

```text
CONFIG_NET_VENDOR_STMICRO=y
CONFIG_STMMAC_ETH=y
CONFIG_STMMAC_PLATFORM=y
CONFIG_DWMAC_ROCKCHIP=y
CONFIG_PHYLIB=y
CONFIG_MOTORCOMM_PHY=y
```

这证明当前正在运行的 6.18.33 内核中：

- stmmac core 已启用
- stmmac platform 已启用
- Rockchip DWMAC 驱动已启用
- PHYLIB 已启用
- Motorcomm PHY 驱动已启用

## 4. 为什么 `/lib/modules` 里没有 stmmac/dwmac 模块

TTL 检查：

```text
ls /lib/modules/$(uname -r)/kernel/drivers/net/ethernet/stmicro/stmmac
NO_STMMAC_MODULE_DIR_OR_BUILTIN
```

这不是缺失，而是因为相关驱动配置为 `=y`，已经内建进内核，不会生成 `.ko` 模块。

## 5. 运行时 DTB 状态

TTL 检查：

```text
DT ethernet status: disabled
DT ethernet compatible: rockchip,rk3399-gmac
DT ethernet phy-mode: rgmii
DT ethernet clock_in_out: input
```

并且：

```text
find /sys/bus/platform/drivers -iname '*dwmac*'
/sys/bus/platform/drivers/rk_gmac-dwmac

find /sys/bus/platform/devices -iname '*ethernet*'
# 无输出
```

解释：

- 驱动存在：`/sys/bus/platform/drivers/rk_gmac-dwmac`
- 设备不存在：因为 DTB 节点 disabled，内核没有实例化 `fe300000.ethernet`

## 6. 当前 eth0 状态

TTL 检查：

```text
ls /sys/class/net
lo

ip -brief link
lo UNKNOWN 00:00:00:00:00:00 <LOOPBACK,UP,LOWER_UP>

ethtool eth0
No data available
```

所以当前没有 `eth0` 是预期结果。

## 7. 与历史文档中“驱动缺失”的关系

`FLASH_GUIDE_6.18.33.md` 中写过：

```text
此镜像缺少 dwmac-rockchip 驱动，以太网将无法工作
```

这个判断适用于早期构建/当时的镜像预期，但已经被当前编译日志与运行时配置推翻。

当前事实是：

```text
CONFIG_DWMAC_ROCKCHIP=y
```

所以现在的有线网卡问题不是驱动缺失，而是人为禁用了 DTB GMAC 节点以保证系统稳定启动。

## 8. 是否应该重新启用 GMAC

技术上可以把 DTB 中：

```text
/ethernet@fe300000/status = "okay"
```

但不建议在当前稳定系统上直接启用。原因：此前多轮测试已经证明板载 GMAC 存在严重问题：

- 主线 stmmac probe/open 会触发 DMA reset
- DMA Bus_Mode SWR 位卡死
- 系统可能重新死机或卡死

当前稳定基线应继续保持：

```text
/ethernet@fe300000/status = disabled
```

如果要重新测试 GMAC，应先备份当前可启动 DTB，再单独做实验版本。
