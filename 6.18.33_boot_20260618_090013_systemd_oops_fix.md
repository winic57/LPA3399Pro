## 1. 新日志结论

上一轮 `root=PARTUUID` 修改已成功：

```text

VFS: Mounted root (ext4 filesystem) on device 179:98.
Run /sbin/init as init process

```

当前失败点变为 systemd 早期执行 generators 阶段触发内核 Oops：

```text
Unable to handle kernel paging request ...
CPU: 4 PID: 138 Comm: systemd-system-
pc : __handle_mm_fault+0x31c/0x184c
systemd[1]: Failed to fork off sandboxing environment for executing generators: Protocol error
systemd[1]: Freezing execution.
```

`systemd-system-` 很可能是 `systemd-system-update-generator` 等 generator 进程名截断。systemd 257 在系统管理器启动时会 fork 一个带 mount namespace/private tmp 的 sandbox 来并行执行 generators；该阶段触发 kernel Oops 后，systemd 收到 `EPROTO` 并冻结。

## 2. 本次 SD 卡修改

### 2.1 禁用 systemd generators

已备份并清空：

```text
/usr/lib/systemd/system-generators
```

备份目录：

```text
/usr/lib/systemd/system-generators.disabled_20260618_091558
/usr/lib/systemd/system-generators.disabled_active
```

目的：让 systemd 启动时 `generator_path_any()` 为 false，跳过 generator sandbox fork/执行阶段，绕过当前内核 Oops。

### 2.2 显式启用登录控制台

因为禁用了 `systemd-getty-generator`，已手工创建：

```text
/etc/systemd/system/multi-user.target.wants/serial-getty@ttyS2.service
/etc/systemd/system/serial-getty@ttyS2.service.d/autologin.conf
/etc/systemd/system/getty.target.wants/getty@tty1.service
/etc/systemd/system/getty@tty1.service.d/autologin.conf
```

串口使用 `ttyS2 1500000`，root autologin。

### 2.3 rootfs 稳定化

- `/etc/fstab` root 项改为 `PARTUUID=61ec8aeb-3d1a-48fa-a9da-54d744ed8bdf`
- `commit=600` 改为 `commit=60`
- 禁用 `fstrim.timer` / `e2scrub_all.timer`
- mask `NetworkManager-wait-online.service`
- 删除 `/root/.not_logged_in_yet`

## 3. 预期

## 4. 回退方法

如需回退 generator 禁用：

```bash
mount /dev/sdc2 /mnt/sdc_root
rm -rf /mnt/sdc_root/usr/lib/systemd/system-generators/*
cp -a /mnt/sdc_root/usr/lib/systemd/system-generators.disabled_20260618_091558/.       /mnt/sdc_root/usr/lib/systemd/system-generators/
umount /mnt/sdc_root
```
