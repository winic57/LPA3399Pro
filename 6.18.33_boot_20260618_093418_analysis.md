# 6.18.33 boot_20260618_093418 分析

## 结论

`maxcpus=4` 生效后，本轮没有再出现 CPU4/CPU5 Oops 或 kernel panic。系统已经推进到 multi-user 阶段：

- rootfs 挂载成功
- systemd 启动成功
- sysinit/basic/network target 达成
- NetworkManager / ssh / systemd-logind 等服务开始或已启动

这验证了上一轮判断：CPU4/CPU5 A72 big cores 在当前主线 DTB/电源/OPP 组合下不稳定，禁用 big cores 是当前可用启动基线。

## 当前剩余问题

日志末尾停在：

```text
Starting plymouth-quit-wait.service - Hold until boot process finishes up...
Starting plymouth-quit.service - Terminate Plymouth Boot Screen...
```

同时早期有：

```text
Expecting device dev-ttyAML0.device
Expecting device dev-ttyS2.device
```

说明系统基本启动，但串口登录尚未稳定出现。可能原因：

1. `plymouth` 仍在接管/污染控制台输出；
2. 之前为降低 Oops 风险 mask 了 `systemd-udev-trigger.service`，导致 systemd device unit 冷插拔不完整，`serial-getty@ttyS2.service` 等待 `dev-ttyS2.device`；
3. 镜像里仍残留错误的 `ttyAML0` getty 依赖，应清理。

## 下一步建议

SD 卡插回宿主机后执行：

1. boot 参数追加：
   - `plymouth.enable=0`

2. ROOTFS 上 mask：
   - `plymouth-start.service`
   - `plymouth-quit.service`
   - `plymouth-quit-wait.service`
   - `plymouth-read-write.service`

3. 恢复：
   - `systemd-udev-trigger.service`

4. 清理错误串口：
   - 删除或 mask `serial-getty@ttyAML0.service`

5. 保留：
   - `maxcpus=4`
   - GMAC disabled
   - PCIe okay
   - generators disabled
   - drm/configfs/efi_pstore 相关 mask
