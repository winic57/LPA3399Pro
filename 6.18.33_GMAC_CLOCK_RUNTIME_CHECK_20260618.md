## 3. 观察

虽然本轮把 `clock_in_out` 改成了 `output`，但 `assigned-clock-parents` 仍指向 `<0x20>`，即当前 DTB 中的 `clkin_gmac`。这会造成“配置语义不一致”：

- `clock_in_out = output` 表示 MAC/CRU 侧输出参考时钟；
- 但 assigned parent 仍指定外部 `clkin_gmac` 作为父时钟。

因此本轮 `clock_in_out=output` 失败，不能完全排除 output 模式，需要进一步确认：

1. `<0x20>` 对应的 phandle 是否确实是 `clkin_gmac`；
2. 如果测试 output 模式，应把 `assigned-clock-parents` 改成内部 PLL/时钟源；
3. 如果测试 input 模式，则需要确认外部 PHY 125MHz 是否稳定，并且 DMA reset 前 MAC 相关 clock gate 是否完整。

## 4. 当前失败仍为

## 5. 下一步建议

下一步不建议继续盲改 reset/delay。应先 decompile DTB，确认 clock phandle 和 CRU clock ID：

- `/ethernet@fe300000/assigned-clocks`
- `/ethernet@fe300000/assigned-clock-parents`
- `clkin_gmac` 节点 phandle
- 可用内部 PLL phandle，如 `cpll`、`gpll`

然后再做一致的 output 模式测试，或者转入 `dwmac-rk.c` 驱动补丁。
