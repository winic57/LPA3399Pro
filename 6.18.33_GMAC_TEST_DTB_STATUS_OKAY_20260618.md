# 6.18.33 GMAC 独立测试 DTB 修改记录

日期: 2026-06-18
目标: 复制独立 `gmac-test.dtb`，仅测试 `/ethernet@fe300000/status = "okay"`

## 修改内容

SD 卡 boot 分区挂载点:

```text
/mnt/sdc_boot
```

原稳定 DTB:

```text
/mnt/sdc_boot/dtb/rockchip/rk3399pro-neardi-linux-lc110-base.dtb
```

新测试 DTB:

```text
/mnt/sdc_boot/dtb/rockchip/gmac-test.dtb
```

执行修改:

```bash
cp -a /mnt/sdc_boot/dtb/rockchip/rk3399pro-neardi-linux-lc110-base.dtb \
      /mnt/sdc_boot/dtb/rockchip/gmac-test.dtb
fdtput -t s /mnt/sdc_boot/dtb/rockchip/gmac-test.dtb \
      /ethernet@fe300000 status okay
```

验证结果:

```text
fdtget -t s /mnt/sdc_boot/dtb/rockchip/gmac-test.dtb /ethernet@fe300000 status
okay
```

## extlinux 修改

已将启动 DTB 从稳定版改为测试版:

```diff
- FDT /dtb/rockchip/rk3399pro-neardi-linux-lc110-base.dtb
+ FDT /dtb/rockchip/gmac-test.dtb
```

备份文件:

```text
/mnt/sdc_boot/extlinux/extlinux.conf.gmac-test.bak
```

## 重要说明

本次只改了 GMAC status：

```dts
/ethernet@fe300000/status = "okay";
```

没有修改：

- `snps,reset-gpio`
- `rx_delay`
- `tx_delay`
- `assigned-clock-parents`
- 其他 DTB 节点

当前测试 DTB 仍保留原来的：

```text
snps,reset-gpio = gpio3 PB7 active-low
rx_delay = 0x11
tx_delay = 0x28
```

## 下一步 TTL 观察点

启动后重点查看：

```bash
dmesg | grep -iE 'gmac|stmmac|dwmac|eth0|phy|motorcomm|yt8521'
ip link
```

如果能进入系统但 eth0 未 up，再手工测试：

```bash
timeout 5 ip link set eth0 up
dmesg | tail -n 120
```

如果出现 DMA reset/SWR timeout 或死机，回退 extlinux 到稳定 DTB：

```bash
cp -a /mnt/sdc_boot/extlinux/extlinux.conf.gmac-test.bak \
      /mnt/sdc_boot/extlinux/extlinux.conf
```
